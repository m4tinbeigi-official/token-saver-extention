# Token Saver — اپلیکیشن دسکتاپ (Windows / macOS)

ابزار سلف‌سرویس Token Saver: با چند کلیک، پروژه‌ات را اسکن می‌کند، چند سؤال می‌پرسد و بر اساس پاسخ‌ها فایل‌های کانفیگ بهینه‌سازی توکن را برای agentهای کدنویسی (Claude Code، Cursor، Codex CLI، Gemini CLI) می‌سازد.

ساخته‌شده با **Electron** تا روی ویندوز و مک با یک فایل نصبی اجرا شود.

## اجرا در حالت توسعه

```bash
cd desktop
npm install
npm start
```

## تست منطق هسته (بدون Electron)

```bash
npm test
```

## ساخت فایل نصبی

> توجه: خروجی مک باید روی macOS و خروجی ویندوز ترجیحاً روی ویندوز ساخته شود.

```bash
# ویندوز (.exe نصبی – NSIS)
npm run dist:win

# مک (.dmg برای Intel و Apple Silicon)
npm run dist:mac
```

خروجی‌ها در پوشه‌ی `release/` ساخته می‌شوند.

### build یک‌کلیکی

- مک: فایل `build-all.command` را در Finder دابل‌کلیک کن (مک را native و ویندوز را با Docker/Wine می‌سازد).
- ویندوز: فایل `build-all.bat` را دابل‌کلیک کن.

## آیکون و assets

آیکون اپ و بک‌گراند DMG با اسکریپت `make-assets.py` ساخته می‌شوند (خروجی در `build/`):

```bash
python3 make-assets.py   # نیازمند Pillow
```

تولید می‌کند: `icon.png` (1024)، `icon.ico` (ویندوز)، `icon.icns` (مک)، و `dmg-background.png`. electron-builder این‌ها را خودکار برمی‌دارد.

## نصب روی مک (نکته مهم برای اپ unsigned)

چون اپ هنوز با Apple Developer ID امضا/notarize نشده، مک ممکن است هنگام باز کردن خطای «is damaged» یا «unidentified developer» بدهد. دو راه:

1. **راست‌کلیک روی اپ → Open** (به‌جای دابل‌کلیک)، سپس در دیالوگ Open را بزن.
2. یا فایل `fix-mac-run.command` را دابل‌کلیک کن — که نشان quarantine را پاک و اپ را ad-hoc امضا می‌کند:

```bash
xattr -cr "/Applications/Token Saver.app"
codesign --force --deep --sign - "/Applications/Token Saver.app"
```

برای حذف کامل این هشدارها، در آینده می‌توان اپ را با Apple Developer ID امضا و notarize کرد.

## جریان کار اپ

۱. **انتخاب پروژه** — پوشه‌ی ریشه انتخاب می‌شود؛ Stack، دستور تست/build و کانفیگ‌های موجود تشخیص داده می‌شوند (فقط خواندن).
۲. **سؤال‌های فنی + اطلاعات کاربر** — agentها، خروجی‌های پرنویز، اندازه ریپو، اولویت و provider؛ به‌علاوه نام/ایمیل اختیاری که (best-effort) روی سرور ذخیره می‌شود.
۳. **ابزارها** — فهرست ابزارهای متن‌باز با توضیح کامل و دکمه‌ی **نصب خودکار** (با نمایش خروجی زنده). اولین ابزار: `codebase-memory-mcp`.
۴. **کانفیگ** — پیش‌نمایش فایل‌های کانفیگ + اعمال (با بکاپ `.tokensaver.bak`).

## افزودن ابزار جدید (توسعه‌پذیری)

کافی است یک شیء جدید به آرایه‌ی `TOOLS` در `tools-registry.js` اضافه کنی؛ UI و نصب‌کننده خودکار آن را نمایش و اجرا می‌کنند. هر ابزار این فیلدها را دارد: `id, name, tagline, description, howItWorks, claims[], notes[], afterInstall, repo, install{ mac, win, linux }`. هر `install` می‌تواند `type: 'shell'` (اجرای خودکار با bash) یا `type: 'manual'` (فقط نمایش دستور) باشد.

## ذخیره روی سرور

اطلاعات کاربر با `POST` به `SERVER_ENDPOINT` (در `main.js`) فرستاده می‌شود. اگر سرور در دسترس نباشد، با timeout پنج‌ثانیه‌ای **بی‌صدا رد می‌شود** و جریان کار متوقف نمی‌گردد. آدرس را در `main.js` به endpoint واقعی خودت تغییر بده.

## امنیت

- هیچ فایلی بدون مرحله‌ی پیش‌نمایش و دکمه‌ی «اعمال» نوشته نمی‌شود.
- نوشتن فقط داخل پوشه‌ی انتخاب‌شده مجاز است (مسیرهای خارج از پروژه رد می‌شوند).
- نصب ابزار فقط با **کلیک صریح کاربر** و با **نمایش کامل خروجی** اجرا می‌شود؛ دستور نصب همیشه قابل مشاهده و کپی است.
- روی ویندوز، نصب‌کننده‌های مبتنی بر `bash` به WSL یا Git Bash نیاز دارند؛ در نبودشان دستور برای اجرای دستی نمایش داده می‌شود.
