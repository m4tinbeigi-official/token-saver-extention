# Token Saver — اپلیکیشن دسکتاپ (Windows / macOS)

ابزار سلف‌سرویس Token Saver: با چند کلیک، پروژه‌ات را اسکن می‌کند، چند سؤال می‌پرسد و بر اساس پاسخ‌ها فایل‌های کانفیگ بهینه‌سازی توکن را برای agentهای کدنویسی (Claude Code، Cursor، Codex CLI، Gemini CLI) می‌سازد.

ساخته‌شده با **Electron** تا روی ویندوز و مک با یک فایل نصبی اجرا شود.

## اجرا در حالت توسعه

```bash
cd desktop
npm install
npm start
```

## تست منطق هسته (بدون Electron)

```bash
npm test
```

## ساخت فایل نصبی

> توجه: خروجی مک باید روی macOS و خروجی ویندوز ترجیحاً روی ویندوز ساخته شود.

```bash
# ویندوز (.exe نصبی – NSIS)
npm run dist:win

# مک (.dmg برای Intel و Apple Silicon)
npm run dist:mac
```

خروجی‌ها در پوشه‌ی `release/` ساخته می‌شوند.

## جریان کار اپ

۱. **انتخاب پروژه** — پوشه‌ی ریشه انتخاب می‌شود؛ Stack، دستور تست/build و کانفیگ‌های موجود تشخیص داده می‌شوند (فقط خواندن).
۲. **سؤال‌ها** — agentها، خروجی‌های پرنویز، اندازه ریپو، اولویت (هزینه/کیفیت) و provider.
۳. **پیش‌نمایش** — همه فایل‌هایی که ساخته می‌شوند با محتوای کامل + پیشنهادها + دستورهای نصب ابزارها.
۴. **اعمال** — فایل‌ها داخل پروژه نوشته می‌شوند؛ از فایل‌های موجود نسخه‌ی پشتیبان `.tokensaver.bak` ساخته می‌شود.

## امنیت

- هیچ فایلی بدون مرحله‌ی پیش‌نمایش و دکمه‌ی «اعمال» نوشته نمی‌شود.
- نوشتن فقط داخل پوشه‌ی انتخاب‌شده مجاز است (مسیرهای خارج از پروژه رد می‌شوند).
- ابزارهای متن‌باز **به‌صورت خودکار نصب نمی‌شوند**؛ فقط دستور نصب نمایش داده می‌شود تا خودت اجرا کنی.

## معماری

| فایل | نقش |
| --- | --- |
| `main.js` | پراسس اصلی Electron + هندلرهای IPC (انتخاب پوشه، اسکن، پیش‌نمایش، نوشتن) |
| `preload.js` | پل امن بین UI و سیستم (contextIsolation) |
| `config-engine.js` | منطق خالص تولید کانفیگ — قابل تست بدون Electron |
| `src/index.html` · `styles.css` · `renderer.js` | رابط کاربری ویزارد |
| `test-engine.js` | تست‌های Node برای موتور کانفیگ |
